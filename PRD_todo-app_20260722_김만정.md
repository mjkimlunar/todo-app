# PRD — Todo 앱 (가족/팀 공유 + 관리자)

- 프로젝트: todo-app
- 작성일: 2026-07-22
- 작성자: 김만정
- 개정: 2026-07-22 — 회원가입/로그인, 모임 관리자, 이메일 초대 추가 (v1의 "진짜
  로그인이 아니다" 한계를 해소)

## 한 줄 개요

가족·팀 구성원들이 계정으로 로그인해 모임에 참여하고, 같은 할 일 목록을 보고
관리하는 Todo 앱. 모임을 만든 사람이 관리자가 되어 이메일로 새 구성원을 초대할
수 있다.

## 해결하려는 문제

집안일이나 팀 업무(빨래, 장보기, 아이 픽업, 업무 분담 등)를 여러 명이 나눠서
하는데, 각자 머릿속으로만 기억하거나 메신저로 흩어져서 이야기하다 보니 "이거
누가 하기로 했지?"가 자주 발생한다. 계정으로 로그인해 모임에 참여하면 같은
목록을 보고 편집할 수 있고, 관리자는 이메일 한 통으로 새 구성원을 초대해 이
문제를 줄인다.

## 타겟 사용자

같은 집 또는 같은 팀의 구성원 2~5명. 기술적으로 능숙하지 않아도 이메일
회원가입 한 번과 초대 이메일의 코드/링크만으로 바로 참여할 수 있어야 한다.

## 핵심 기능

- 회원가입 / 로그인 (이메일 인증 포함)
- 할 일 추가 / 목록 보기 / 완료 표시 / 삭제 (CRUD)
- 마감일, 우선순위(낮음/보통/높음), 태그
- 제목 검색, 태그 필터, "오늘 할 일만 보기"
- **모임 공유**: 모임을 새로 만들거나 기존 초대 코드로 참여. 로그인한 계정이
  그 모임의 구성원이어야만 목록을 보고 편집할 수 있다.
- **관리자 & 이메일 초대**: 모임을 만든 사람이 그 모임의 관리자가 된다. 관리자는
  이메일 주소를 입력해 초대 코드와 참여 링크가 담긴 메일을 보낼 수 있다
  (Resend 이용). 초대 코드로 참여한 사람은 일반 구성원(member)이 된다.

## 기술 구조

- 하네스: Claude Code (터미널형)
- 프론트엔드: 순수 HTML/CSS/JS (빌드 과정 없음). Supabase에는 직접 접근하지
  않고 항상 Express `/api/*`를 거침 (로그인/가입도 동일)
- 백엔드: Node.js + Express, `/api/*` REST 엔드포인트
- 인증: Supabase Auth (이메일/비밀번호, 이메일 인증). Express가 `access_token`을
  검증하고, `household_members` 테이블로 모임 소속 여부와 역할(admin/member)을
  확인한다
- 이메일 발송: Resend (관리자의 초대 메일 전용). Supabase Auth 자체의 가입 인증
  메일은 Supabase 대시보드에서 Resend SMTP를 연결해 발송한다
- 데이터베이스: Supabase(Postgres). RLS 활성화, 정책 없음 — 서버의 service_role
  키(`.env`, 브라우저에 노출 안 됨)만 접근 가능
- 모임 구분: `households` + `household_members`(역할 포함) 테이블. 브라우저는
  참여한 household id를 localStorage에 저장하고, 모든 API 요청에
  `Authorization: Bearer <access_token>` + `X-Household-Id` 헤더로 실어 보낸다
- 배포: Render.com 무료 웹 서비스

## 한계 (알고 있는 것, 다음 단계로 남겨둠)

- **관리자 위임/강등 기능이 없다.** 모임을 만든 사람만 admin이고, 이후에 다른
  구성원을 admin으로 올리거나 admin을 member로 내릴 수 있는 화면이 없다.
- **초대 코드를 잃어버리면 재발급할 방법이 없다.** (관리자가 재발급하는 기능은
  다음 단계)
- **실시간 동기화는 없다.** 다른 사람이 추가한 항목은 새로고침해야 보인다.
- **CLI(`cli.js`)/MCP 서버(`mcp-server.js`)는 로그인 없이 `.env`의
  `HOUSEHOLD_ID` 하나로 동작한다.** 개발자 본인의 로컬 조회 도구용이며, 여러
  모임을 오가며 조회하려면 매번 `.env` 값을 바꿔야 한다.

## 회고

- 로그인 절차 자체는 간단해 보였는데, 막상 붙이려니 Supabase Auth 설정, Resend
  가입과 API 키 발급, Supabase SMTP 연결처럼 추가로 가입하고 세팅해야 할
  외부 클라우드 서비스와 설정 항목이 생각보다 많아서 손이 많이 갔다.
- 원래는 가족뿐 아니라 친구, 지인, 회사 동료까지 두루 쓸 수 있는 커뮤니티 같은
  앱으로 키우고 싶었는데, 그러려면 어떤 기능이 필요한지 정의하는 것부터
  막막했다.
