# PRD — Todo 앱 (가족 공유)

- 프로젝트: todo-app
- 작성일: 2026-07-22
- 작성자: 김만정

## 한 줄 개요

가족 구성원들이 초대 코드 하나로 같은 할 일 목록을 보고 관리하는, 내 컴퓨터에서
바로 돌아가는 Todo 앱.

## 해결하려는 문제

집안일(빨래, 장보기, 아이 픽업 등)을 가족끼리 나눠서 하는데, 각자 머릿속으로만
기억하거나 메신저로 흩어져서 이야기하다 보니 "이거 누가 하기로 했지?"가 자주
발생한다. 로그인 없이 코드 하나만 공유하면 바로 같은 목록을 볼 수 있는 가장
가벼운 형태로 이 문제를 줄인다.

## 타겟 사용자

같은 집에 사는 가족 구성원 2~5명. 기술적으로 능숙하지 않아도 코드 하나만
입력하면 바로 쓸 수 있어야 한다.

## 핵심 기능

- 할 일 추가 / 목록 보기 / 완료 표시 / 삭제 (CRUD)
- 마감일, 우선순위(낮음/보통/높음), 태그
- 제목 검색, 태그 필터, "오늘 할 일만 보기"
- **가족 공유**: 초대 코드로 새 가족 만들기 / 기존 코드로 참여하기. 코드를 아는
  사람은 로그인 없이 그 가족의 목록을 보고 편집할 수 있다.

## 기술 구조

- 하네스: Claude Code (터미널형)
- 프론트엔드: 순수 HTML/CSS/JS (빌드 과정 없음)
- 백엔드: Node.js + Express, `/api/*` REST 엔드포인트
- 데이터베이스: Supabase(Postgres). RLS 활성화, 정책 없음 — 서버의 service_role
  키(`.env`, 브라우저에 노출 안 됨)만 접근 가능
- 가족 구분: `households` 테이블 + 초대 코드. 브라우저는 참여한 household id를
  localStorage에 저장하고, 모든 API 요청에 `X-Household-Id` 헤더로 실어 보낸다
- 배포: Render.com 무료 웹 서비스

## 한계 (알고 있는 것, 다음 단계로 남겨둠)

- **진짜 로그인이 아니다.** 초대 코드만 알면 누구나 그 가족의 목록에 접근할 수
  있다. 가족 단위의 낮은 보안 위험을 전제로 한 첫 번째 버전이며, 실제 계정별
  권한이 필요해지면 Supabase Auth + `auth.uid()` 기반 RLS로 확장할 수 있다.
- 초대 코드를 잃어버리면 그 가족 목록에 다시 들어갈 방법이 없다(재발급 기능 없음).
- 실시간 동기화는 없다 — 다른 사람이 추가한 항목은 새로고침해야 보인다.
