# Todo 정리 (마감 지난 완료 항목 삭제)

이 프로젝트에서 사용자가 주기적으로 반복하는 정리 작업이다. 완료 처리는
했지만 마감일이 지난 항목들은 더 이상 볼 필요가 없으므로 삭제한다. 이미
만들어진 `cli.js`(조회 전용)와 `server.js`의 REST API(삭제 실행)를 그대로
재사용한다 — 이 작업을 위해 새 DB 로직을 만들 필요는 없다.

## 절차

### 1. 정리 대상 조회 (읽기 전용, 안전)

```bash
node --env-file=.env cli.js list --overdue-completed --json
```

완료(`is_completed: true`)되었고 `due_date`가 오늘보다 이전인 항목만 JSON
배열로 나온다. 프로젝트 루트(`todo-app/`)에서 실행해야 한다.

### 2. 대상이 없으면 종료

배열이 비어 있으면(`[]`) "정리할 항목이 없습니다"라고 사용자에게 알리고
끝낸다. 억지로 뭔가를 삭제하려 하지 않는다.

### 3. 대상 목록을 보여주고 확인받기

대상이 있으면 각 항목의 제목·마감일을 사용자에게 보여주고, **삭제해도
되는지 명시적으로 물어본다.** 자동으로 삭제하지 않는다 — 이 확인 단계는
생략 대상이 아니다. 사용자가 일부만 골라 삭제하고 싶어할 수도 있으니,
전체 삭제/부분 삭제 모두 가능하다고 안내한다.

### 4. 삭제 전 체크포인트 확인

`git status`로 커밋되지 않은 변경사항이 있는지 확인한다. 있다면 먼저
커밋할지 사용자에게 물어본다(코드 변경을 안전하게 되돌릴 수 있는 지점을
만들기 위함 — Supabase의 실제 데이터 자체는 git으로 되돌릴 수 없다는 점을
사용자가 알고 있어야 한다).

### 5. 로컬 서버 확인 후 삭제 실행

`http://localhost:3000`에 로컬 서버가 떠 있는지 확인한다(안 떠 있으면
`npm run dev`로 띄우도록 안내). 사용자가 확인한 각 항목 id에 대해:

```bash
curl -X DELETE http://localhost:3000/api/todos/<id>
```

성공하면 `204` 응답이 온다.

### 6. 결과 재확인

```bash
node --env-file=.env cli.js summary
```

삭제 전후 개수를 비교해서 사용자에게 알려준다.

## 참고

- 이 스킬은 오직 **완료 + 마감 지남** 항목만 대상으로 한다. 미완료 항목은
  아무리 마감이 지나도 이 스킬로 건드리지 않는다(할 일이 아직 안 끝난
  것이므로 사용자가 직접 판단해야 함).
- `AGENTS.md`에 정리된 규칙(CLI는 읽기 전용, 데이터 변경 전 git 체크포인트)을
  그대로 따른다.
